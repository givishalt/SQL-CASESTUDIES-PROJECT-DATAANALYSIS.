## 📌 Project Overview

This project is a comprehensive **SQL Case Study** focused on analysing sales, profit, marketing spend, and inventory data across multiple US states and product categories. As a **Database Administrator**, the goal is to extract meaningful business insights from three relational tables using SQL queries ranging from basic aggregations to advanced window functions, stored procedures, and user-defined functions.

> **Skills Demonstrated:** SELECT, WHERE, GROUP BY, HAVING, JOIN, UNION, Window Functions, Stored Procedures, UDFs, Transactions, ROLLUP, CASE statements, and String Functions.

---

## 🗃️ Dataset Description

The case study uses **3 tables** with a total of **4,369 records**.

### FactTable — 4,200 rows × 14 columns

| Column | Description |
|---|---|
| `Date` | Date of the transaction |
| `ProductID` | Unique product identifier |
| `Profit` | Profit from the sale |
| `Sales` | Total sales value |
| `Margin` | Profit margin |
| `COGS` | Cost of Goods Sold |
| `Total_Expenses` | Total expenses incurred |
| `Marketing` | Marketing spend |
| `Inventory` | Inventory count |
| `Budget_Profit` | Budgeted profit |
| `Budget_COGS` | Budgeted COGS |
| `Budget_Margin` | Budgeted margin |
| `Budget_Sales` | Budgeted sales |
| `Area_Code` | Geographic area code |

### ProductTable — 13 rows × 4 columns

| Column | Description |
|---|---|
| `Product_Type` | High-level category (Coffee, Tea, etc.) |
| `Product` | Product name |
| `ProductID` | Unique product identifier |
| `Type` | Sub-type (Regular, Decaf, etc.) |

### LocationTable — 156 rows × 4 columns

| Column | Description |
|---|---|
| `Area_Code` | Geographic area code |
| `State` | US State name |
| `Market` | Market region |
| `Market_Size` | Market size classification |

## ✅ Tasks & SQL Solutions

### 🔹 Basic Queries (Q1–Q9)

**Q1. Display the number of states present in the LocationTable**
```sql
SELECT COUNT(DISTINCT State) AS Number_of_States
FROM LocationTable;
```

---

**Q2. How many products are of regular type?**
```sql
SELECT COUNT(*) AS Regular_Products
FROM ProductTable
WHERE Type = 'Regular';
```

---

**Q3. How much spending has been done on marketing of product ID 1?**
```sql
SELECT SUM(Marketing) AS Total_Marketing_Spend
FROM FactTable
WHERE ProductID = 1;
```

---

**Q4. What is the minimum sales of a product?**
```sql
SELECT MIN(Sales) AS Minimum_Sales
FROM FactTable;
```

---

**Q5. Display the max Cost of Goods Sold (COGS)**
```sql
SELECT MAX(COGS) AS Maximum_COGS
FROM FactTable;
```

---

**Q6. Display the details of the product where product type is Coffee**
```sql
SELECT *
FROM ProductTable
WHERE Product_Type = 'Coffee';
```

---

**Q7. Display the details where total expenses are greater than 40**
```sql
SELECT *
FROM FactTable
WHERE Total_Expenses > 40;
```

---

**Q8. What is the average sales in area code 719?**
```sql
SELECT AVG(Sales) AS Avg_Sales
FROM FactTable
WHERE Area_Code = 719;
```

---

**Q9. Find out the total profit generated by Colorado state**
```sql
SELECT SUM(f.Profit) AS Total_Profit
FROM FactTable f
JOIN LocationTable l ON f.Area_Code = l.Area_Code
WHERE l.State = 'Colorado';
```

### 🔹 Aggregation Queries (Q10–Q14)

**Q10. Display the average inventory for each product ID**
```sql
SELECT ProductID, AVG(Inventory) AS Avg_Inventory
FROM FactTable
GROUP BY ProductID;
```

---

**Q11. Display state in a sequential order in the LocationTable**
```sql
SELECT DISTINCT State
FROM LocationTable
ORDER BY State ASC;
```

---

**Q12. Display the average budget where average budget margin > 100**
```sql
SELECT ProductID,
       AVG(Budget_Profit) AS Avg_Budget_Profit,
       AVG(Budget_Margin) AS Avg_Budget_Margin
FROM FactTable
GROUP BY ProductID
HAVING AVG(Budget_Margin) > 100;
```

---

**Q13. What is the total sales done on date 2010-01-01?**
```sql
SELECT SUM(Sales) AS Total_Sales
FROM FactTable
WHERE Date = '2010-01-01';
```

---

**Q14. Display the average total expense of each product ID on an individual date**
```sql
SELECT Date, ProductID, AVG(Total_Expenses) AS Avg_Total_Expenses
FROM FactTable
GROUP BY Date, ProductID
ORDER BY Date, ProductID;
```

---

### 🔹 Joins (Q15–Q18)

**Q15. Display date, productID, product_type, product, sales, profit, state, area_code**
```sql
SELECT
    f.Date,
    f.ProductID,
    p.Product_Type,
    p.Product,
    f.Sales,
    f.Profit,
    l.State,
    f.Area_Code
FROM FactTable f
JOIN ProductTable  p ON f.ProductID   = p.ProductID
JOIN LocationTable l ON f.Area_Code   = l.Area_Code;
```

---

**Q16. Display the rank without any gap to show the sales wise rank (DENSE_RANK)**
```sql
SELECT
    Date,
    ProductID,
    Sales,
    DENSE_RANK() OVER (ORDER BY Sales DESC) AS Sales_Rank
FROM FactTable;
```

---

**Q17. Find the state wise profit and sales**
```sql
SELECT
    l.State,
    SUM(f.Profit) AS Total_Profit,
    SUM(f.Sales)  AS Total_Sales
FROM FactTable f
JOIN LocationTable l ON f.Area_Code = l.Area_Code
GROUP BY l.State
ORDER BY Total_Sales DESC;
```

---

**Q18. Find the state wise profit and sales along with the product name**
```sql
SELECT
    l.State,
    p.Product,
    SUM(f.Profit) AS Total_Profit,
    SUM(f.Sales)  AS Total_Sales
FROM FactTable f
JOIN ProductTable  p ON f.ProductID = p.ProductID
JOIN LocationTable l ON f.Area_Code = l.Area_Code
GROUP BY l.State, p.Product
ORDER BY l.State, Total_Sales DESC;
```

### 🔹 Advanced Queries (Q19–Q24)

**Q19. If there is an increase in sales of 5%, calculate the increased sales**
```sql
SELECT
    Date,
    ProductID,
    Sales                      AS Original_Sales,
    Sales * 1.05               AS Increased_Sales_5_Pct
FROM FactTable;
```

---

**Q20. Find the maximum profit along with the product ID and product type**
```sql
SELECT
    f.ProductID,
    p.Product_Type,
    MAX(f.Profit) AS Max_Profit
FROM FactTable f
JOIN ProductTable p ON f.ProductID = p.ProductID
GROUP BY f.ProductID, p.Product_Type
ORDER BY Max_Profit DESC;
```

---

**Q21. Create a stored procedure to fetch result according to product type**
```sql
CREATE PROCEDURE GetProductByType
    @ProductType VARCHAR(50)
AS
BEGIN
    SELECT *
    FROM ProductTable
    WHERE Product_Type = @ProductType;
END;

-- Execute
EXEC GetProductByType @ProductType = 'Coffee';
```

---

**Q22. If total expenses < 60 then Profit, else Loss**
```sql
SELECT
    Date,
    ProductID,
    Total_Expenses,
    CASE
        WHEN Total_Expenses < 60 THEN 'Profit'
        ELSE 'Loss'
    END AS Expense_Status
FROM FactTable;
```

---

**Q23. Total weekly sales with date and product ID using ROLLUP**
```sql
SELECT
    Date,
    ProductID,
    SUM(Sales) AS Total_Sales
FROM FactTable
GROUP BY ROLLUP (Date, ProductID)
ORDER BY Date, ProductID;
```

---

**Q24. Apply UNION and INTERSECT on tables with Area Code**
```sql
-- UNION
SELECT Area_Code FROM FactTable
UNION
SELECT Area_Code FROM LocationTable;

-- INTERSECT
SELECT Area_Code FROM FactTable
INTERSECT
SELECT Area_Code FROM LocationTable;
```

### 🔹 Functions, Procedures & String Operations (Q25–Q29)

**Q25. Create a user-defined function to fetch product type**
```sql
CREATE FUNCTION GetProductByTypeFn
    (@ProductType VARCHAR(50))
RETURNS TABLE
AS
RETURN
(
    SELECT *
    FROM ProductTable
    WHERE Product_Type = @ProductType
);

-- Usage
SELECT * FROM dbo.GetProductByTypeFn('Coffee');
```

---

**Q26. Change product type from Coffee to Tea where ProductID = 1, then undo it**
```sql
BEGIN TRANSACTION;

UPDATE ProductTable
SET Product_Type = 'Tea'
WHERE ProductID = 1;

-- Undo the change
ROLLBACK TRANSACTION;

-- Verify rollback
SELECT * FROM ProductTable WHERE ProductID = 1;
```

---

**Q27. Display date, product ID and sales where total expenses are between 100 and 200**
```sql
SELECT Date, ProductID, Sales
FROM FactTable
WHERE Total_Expenses BETWEEN 100 AND 200
ORDER BY Date;
```

---

**Q28. Delete records in ProductTable for regular type**
```sql
DELETE FROM ProductTable
WHERE Type = 'Regular';

-- Verify
SELECT * FROM ProductTable;
```

---

**Q29. Display the ASCII value of the fifth character from column Product**
```sql
SELECT
    Product,
    SUBSTRING(Product, 5, 1)        AS Fifth_Character,
    ASCII(SUBSTRING(Product, 5, 1)) AS ASCII_Value
FROM ProductTable;
```
